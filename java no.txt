	This classification governs how threads interact with the Java Virtual Machine (JVM) lifecycle and 
	determine when an application exits.

	User Threads (Foreground Threads):
		 High-priority threads designed to execute core application logic.
		 The JVM will active and refuse to shut down until all user threads finish running. 
		The default main thread is a user thread.

	Daemon Threads (Background Threads): 
		Low-priority threads that provide utility services to user threads. 

		Examples include the Java Garbage Collector or finalizer threads. 
		The JVM terminates abruptly and exits immediately once the 
		last active user thread finishes, disregarding any remaining daemon threads.
------------------------------------------------------------------------------------------------------------------------
  
	Platform Threads: 
		Traditional Java threads that act as thin wrappers around operating 
		system (OS) kernel threads.
		 They carry a heavy memory footprint (usually 1MB for the call stack) 
		and incur high context-switching costs.

	Virtual Threads: 
		Lightweight threads managed entirely by the JVM instead of the OS. 
		Hundreds of thousands of virtual threads can run concurrently on 
		a single platform thread, significantly reducing memory and overhead 
		for I/O-bound workloads.
